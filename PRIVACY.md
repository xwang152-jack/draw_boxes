# Privacy Policy

## Image Bounding Box Plugin Privacy Policy

**Effective Date:** September 2025  
**Version:** 1.0

### Overview

This privacy policy explains how the Image Bounding Box Plugin (hereinafter referred to as "this plugin") collects, uses, stores, and protects user data. We are committed to protecting user privacy and ensuring data security.

### Data Collection

#### Types of Data We Collect

1. **Image Data**
   - Image files provided by users (via URL or Base64 format)
   - Image metadata (dimensions, format, etc.)

2. **Annotation Data**
   - Annotation information provided by users (bounding box coordinates, labels, confidence scores, etc.)
   - Annotation configuration parameters (colors, line width, font size, etc.)

3. **Technical Data**
   - Plugin usage logs
   - Error reports and performance data

#### Data Collection Methods

- Parameters and data actively input by users
- Technical logs automatically generated during plugin operation

### Data Usage

#### Purpose of Data Usage

1. **Core Functionality Implementation**
   - Processing image data to draw annotations
   - Generating annotated result images

2. **Service Improvement**
   - Analyzing usage patterns to optimize performance
   - Identifying and fixing technical issues

3. **Security Assurance**
   - Detecting and preventing abuse
   - Maintaining service stability

#### Data Processing Principles

- **Minimization Principle**: Only collect and process necessary data
- **Purpose Limitation**: Data is only used for declared purposes
- **Transparency**: Clearly explain data processing methods

### Data Storage and Security

#### Storage Methods

1. **Temporary Processing**
   - Image data is temporarily stored in memory during processing
   - Temporary data is immediately cleared after processing completion

2. **Log Data**
   - Technical logs may be retained short-term for troubleshooting
   - Does not include users' sensitive image content

#### Security Measures

1. **Data Transmission Security**
   - Use HTTPS protocol for data transmission
   - Encrypt sensitive data

2. **Access Control**
   - Restrict data access permissions
   - Implement authentication and authorization mechanisms

3. **Data Protection**
   - Regular security audits
   - Timely application of security updates

### Data Sharing and Disclosure

#### Non-Sharing Principle

We do not sell, trade, or transfer user data to third parties, except:

1. **Legal Requirements**
   - Compliance with applicable laws and regulations
   - Response to legitimate requests from government agencies

2. **Security Needs**
   - Protecting our rights and property
   - Preventing fraud and abuse

3. **User Consent**
   - With explicit user consent

#### Service Providers

We may use third-party service providers to support plugin operation. These providers:
- Can only access data necessary to perform services
- Are bound by strict confidentiality agreements
- Must comply with the same privacy protection standards

1. **Right of Access**: Understanding the types of data we collect
2. **Right of Rectification**: Requesting correction of inaccurate data
3. **Right of Erasure**: Requesting deletion of personal data
4. **Right of Restriction**: Limiting the scope of data processing
5. **Right of Objection**: Objecting to specific data processing activities

### Data Retention

#### Retention Period

1. **Image Data**: Deleted immediately after processing completion
2. **Annotation Data**: Deleted immediately after processing completion
3. **Technical Logs**: Retained for a maximum of 30 days
4. **Error Reports**: Retained for a maximum of 90 days

#### Deletion Policy

- Automatic deletion of expired data
- Users can request data deletion at any time
- All user data deleted upon service termination

### Children's Privacy

This plugin is not specifically designed for children under 13 years old. If we discover that we have collected personal information from children, we will immediately delete the relevant data.

### International Data Transfer

- Implement appropriate safeguards
- Obtain necessary user consent
